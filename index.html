奔赴丽水

3.15号那天，我下午给你发信息你就没有回复我，到了晚上也给你发过信息，你还是没有回复我，到晚上8点，9点，期间我一直给你打电话也未接，我一直在给你打电话,但是你的手机仍然是关机状态，我很着急，就不停的给你打电话，发微信、发短信，你也一直没有回复，那天晚上我一直等到3点多，16号那天上午，我感觉你应该是有什么事情，我就加了黄静的微信，问她有没有跟你联系过，她也是说上午还是给你发过信息，下午就没见你回过信息。我想去你公司看一下，但是我不知你公司地址，自从你们公司搬了好几次后，我也不知道确切位置，然后黄静告诉了我的你的微信密码，我登录到你的淘宝账号，看到你们公司的地址，我立即打车去了你们公司，刚进公司那一层，前台告诉我找谁，我说找你们公司，她跟我说你们公司15号下午被公安带走了，我就打了110，110说一会回复我，过一10分钟左右，110那边的人跟我说因为你们公司的人都被带走了，也没有说什么原因，让我等消息。

下午4点左右你用公安的手机给我打电话，我问你为什么昨天没有给你电话，你说打了信号不好，我没有接到，可是我的确没有接到过任何电话。然后你说你们公司老板因为某些被调查，你们员工也要配合调查，晚上要连夜开车8个小时去浙江，让我不用担心，可以上车后你再给我打电话的时候，我听到你哭了，我很难受。

你3.19号下午给你打电话说让我送一些衣服跟钱过去，我看了下午高铁已经没有票了，我买了20号早上8点的从深圳北到温州南的高铁，再从温州南到丽水市区，下午5点才到，到达看完所的时候门卫已经不让进了，只能第二天上午送了。当天晚上我住在丽水学院旁边的酒店，吃了一个炒粉，在网上找了丽水的律师网站，打了一个电话咨询了一下，刚好他们律师事务所就在我住的地方的附件，不到一公里，我骑着单车马上就到了他们的公司，然后把整个事情给他们讲了一下，老板说整个官司需要3万块，我后面表示只需要了解一下点前的情况，后面报价3000，我当即决定委托，我交了钱，律师表示21号早上就去了解情况。
21号一早你委托看守所的人给你打电话，让我委托律师，我回复已经委托律师，我一早给你送钱跟衣服，不过带过去的两件外套都有扣子跟拉链，都是工作人员去掉了内衣的钢圈，让我再去买两件钻石跟秋衣秋裤。
我马上骑着单车到超市买了几件打底的长袖跟短袖跟裤子，超市的外套不多，我又一路逛了女装店，问了店员我只要卫衣，不要带扣子跟拉链的，逛了十几家，终于买到了两件外套，都是最小的码，但是我觉得还是有些大，我在11点左右赶到了丽水看守所，把衣服给你送了进去。
但是工作人员说衣服不能马上送到，至少也要第二天才能到你手上，期间还要经过检查。然后我跟律师碰了面，律师表示第一个阶段也是拘留阶段的费用是1万块，总共3个阶段，可以分阶段委托，我表示同意，确定委托后我付了1万块，包含了之前的3000，那天下着大雨，我就坐高铁回来了，后面一直在跟律师问你的情况。
得知你的情况良好，我也就放心了，律师表示根据你所说的情况在一个月的拘留期后取保的机率很大，所以我在家等你早点回来。

通过VuePress在github pages搭建个人博客

官网： https://vuepress.docschina.org

vuepress 是vue全家桶中的一员，可以用于搭建静态博客。

全局安装

如果你只是想随便用下 VuePress，你可以在全局安装它：
yarn global add vuepress # 或 npm install -g vuepress

创建一个 markdown 文件

echo ‘# Hello VuePress’ > README.md

开始编写文档

vuepress dev

构建

vuepress build

在已有项目中安装

如果你想要在一个已有项目中维护文档，就应该将 VuePress 安装为本地依赖。此设置还允许你使用 CI 或 Netlify 服务，在推送时自动部署。

安装为本地依赖项

yarn add -D vuepress # 或 npm install -D vuepress

创建一个 docs 目录

mkdir docs

创建一个 markdown 文件

echo ‘# Hello VuePress’ > docs/README.md

警告
当将 VuePress 安装到以 webpack 3.x 作为依赖项的已有项目中时，推荐使用Yarn 而不是 npm。因为在这种情况下，Npm 无法生成正确的依赖关系树。

然后，给 package.json 添加一些 scripts 脚本：

{
“scripts”: {
“docs:dev”: “vuepress dev docs”,
“docs:build”: “vuepress build docs”
}
}
你现在就可以开始编写文档了：

yarn docs:dev # 或 npm run docs:dev
要生成静态资源，请运行：

yarn docs:build # 或 npm run docs:build
默认情况下，构建的文件会位于 .vuepress/dist 中，该文件可以通过 .vuepress/config.js 中的 dest 字段进行配置。构建的文件可以部署到任何静态文件服务器。关于如何部署到一些常用服务，请参考 部署指南。

使用mpvue开发小程序设置全局变量并适配iphoneX

1、设置全局变量：

我们创建好mpvue项目之后，找到src/main.js打开在const app = new Vue(App) 之前添加后面添加一行代码:

Vue.prototype.globalData = getApp().globalData

这行代码的意思就是，将小程序的全局变量，挂载在Vue的原型上，这样我们在不同的页面就都可以获取到全局数据了

不同页面使用 this.globalData获取或者添加修改既可

比方说我需要存储用户一些基本信息可以这样

this.globalData.userInfo = {name:aaa}

不同页面获取方式一样

console.log(this.globalData.userInfo.name)

修改

this.globalData.userInfo.name = ‘bbb’

踩坑注意：在小程序启动的时候，不同页面中vue周期函数created中的代码都会执行一遍。所以vue中created周期函数中尽量不要请求太多接口。而是放在小程序的周期函数onLoad中。

2、适配iphoneX:

微信小程序在适配iphone x 的时候 底部的横线会出现遮挡这时候就要处理下,一般适用fixed定位，固定底部的元素。

在需要引用的页面js文件中onload方法里获取全局变量

let isIphoneX = app.globalData.isIphoneX;

this.isIphoneX: isIphoneX

在 页面文件中 进行css的逻辑判断

css自己加个向下的距离，或者定位看个人需求就可以

.isIphoneX-class{

position:fixed

bottom: 17px

}

使用mpvue框架开发微信小程序

mpvue官网：http://mpvue.com/mpvue/

通过 Vue.js 命令行工具 vue-cli，你只需在终端窗口输入几条简单命令，即可快速创建和启动一个带热重载、保存时静态检查、内置代码构建功能的小程序项目：

全局安装 vue-cli

$ npm install –global vue-cli

创建一个基于 mpvue-quickstart 模板的新项目

$ vue init mpvue/mpvue-quickstart my-project

安装依赖

$ cd my-project
$ npm install # 启动构建 $ npm run dev
接下来，你只需要启动微信开发者工具，引入dist目录即可预览到你的第一个 mpvue 小程序。

mpvue使用vue的语法,组件方面既可以使用官网小程序的组件，也可以自己写vue组件。

第一次使用该框架写小程序的时候需要注意：

1.可以在onload里面对页面进行初始化：

onLoad(res) {
// res.id 是上个页面在链接在链接后添加的参数，比如 /order-detail/main?id=234
wx.setNavigationBarTitle({
title: “产品详情” // 设置当前页面标题
})
}
};
2.页面跳转有两种方式，可以使用navigateTo，可以直接在a链接里面写直跳转的地址

let url = “./order-detail/main”;
wx.navigateTo({ url });

  1. 有时候改变数组或者对象的值后视图不更新，请使用vue中的this.$set() 方法

4.使用官方的选择地区组件的时候可能有些不一样：



所在地区：NaN

methods里面的方法：

bindRegionChange(e) {
this.area = e.target.value
}

有关使用 Hexo 和 GitHub 搭建博客，出现 hexo -d 报错如何解决？

hexo 在提交代码的时候发现报错，提示could not read Username for ‘GitHub

nothing to commit, working directory clean
bash: /dev/tty: No such device or address
error: failed to execute prompt script (exit code 1)
fatal: could not read Username for 'GitHub · Where software is built': No error
FATAL Something's wrong. Maybe you can find the solution here: Troubleshooting
Error: bash: /dev/tty: No such device or address
error: failed to execute prompt script (exit code 1)
fatal: could not read Username for 'GitHub · Where software is built': No error

网上查了一下，发现是因为提交的时候没有传账号，解决办法：
假如如果你的博客地址是这样： myblog.github.io
那么改一下_config.yml 文件里面的提交地址，加上name 和密码
repo: myblog:password@github.com/myblog/myblog.github.io.git

注意：
myblog：这个不是是登录的邮箱，而是你的blog地址
password: 登录密码
